# Design Intelligence Operating Model

Date: 2026-04-24

## Mục Tiêu

Chuẩn hóa cách lớp `Design Intelligence` hoạt động cùng `Flatsome Native Core`.

## Mô Hình 2 Tầng

### Tầng 1: Flatsome Core

Trả lời:

- dùng shortcode nào
- dùng element nào
- dựng section ra sao
- làm sao để mở lại bằng UX Builder

### Tầng 2: Design Intelligence

Trả lời:

- trang này nên mang cảm giác gì
- nhịp bố cục nào hợp nhất
- typography mood nào đúng
- spacing rhythm nào đúng
- motion profile nào đúng
- anti-pattern nào phải tránh

## Công Thức Hoạt Động

`Brief -> Chọn Mode -> Design Intelligence -> Dịch sang Flatsome Core -> Page Plan -> Build -> Audit`

## Luật Cứng

1. `Design Intelligence` không được tự mình quyết định cấu trúc native.
2. Mọi đề xuất đều phải đi qua bước dịch sang `Flatsome Native`.
3. `Flatsome Core` luôn có quyền phủ quyết cuối cùng nếu một ý tưởng không native đủ tốt.
4. Nếu user không yêu cầu sáng tạo, mặc định ưu tiên mode `Flatsome Default`.

## Ba Cách Dùng

### 1. Default

Chỉ dùng `Flatsome Core`.

Dùng khi:

- cần nhanh
- cần ổn định
- brief bình thường

### 2. Assist

`Design Intelligence` đưa ra 2-3 hướng.

Dùng khi:

- user muốn gợi ý
- cần khác hơn các page gần nhất
- muốn nâng độ chuyên nghiệp

### 3. Directed Creative

`Design Intelligence` được bật mạnh.

Dùng khi:

- user nói rõ muốn sáng tạo
- user muốn theo design language cụ thể
- page là showcase / campaign / homepage đặc biệt

## Đầu Ra Của Lớp Này

Mỗi lần dùng `Design Intelligence`, nên chốt tối thiểu:

- style family
- pattern family
- typography mood
- spacing rhythm
- motion profile
- anti-patterns cần tránh

## Kết Luận

Đây là lớp giúp hệ:

- nghĩ rộng hơn
- nhưng vẫn build gọn hơn
- và không đánh mất tính native.
